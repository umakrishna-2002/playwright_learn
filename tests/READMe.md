API Testing: A software testing type that validates Application Programming Interfaces (APIs) to ensure they meet performance, functionality, security, and reliability expectations.

Data Driven Testing: Testing the Code (API) against the multiple data scenario's and Playwright dynamically executes the test once for each data

Why to use Data Driven Testing?

- Write the request logic once and run it against 100 different inputs or scenario's.
- Coverage: Easily add new edge cases (e.g., special characters, long strings, null values) just by adding a new line to your JSON file
- Playwright treats every item in the loop as an independent test, making it clear exactly which data set failed if something breaks.
